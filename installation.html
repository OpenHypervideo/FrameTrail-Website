Server Setup (Recommended)

Installing FrameTrail on a server gives you the full feature set including multi-user collaboration, file uploads, and media processing.

  1. Download the latest release ZIP (or git clone https://github.com/OpenHypervideo/FrameTrail) and extract it to your web server directory.
  2. Set the correct permissions so your web server can write to the directory:
    chmod -R 775 frametrail/ && chown -R www-data:www-data frametrail/
  3. Open the directory in your browser and follow the Setup Wizard to create an administrator account. FrameTrail will automatically create the _data/ folder.
  4. For a quick local test without Apache, run php -S localhost:8080 from the extracted folder and open http://localhost:8080.

nginx users: Add a deny rule for the _data/ directory to prevent direct access to your data files. The included .htaccess handles this automatically for Apache.

Local Folder Setup

  1. Download the latest release ZIP and extract it anywhere on your computer.
  2. Open index.html directly in Chrome or Edge.
  3. When prompted, select an existing _data folder or create a new one. FrameTrail will use this folder to store all your data.
  4. Enter a display name to identify yourself — no account or password is needed in this mode.

That's it! You might want to read the Getting Started Tutorial now.

Updates

To update your FrameTrail instance, download the latest release and copy / replace it on your server. Make sure you do not overwrite the _data/ folder — it contains all your accounts, hypervideos, and project data.

Data & Backup

All data is stored as JSON files in the _data/ directory — there is no database. To back up your project, simply copy the entire _data/ folder. To migrate to a new server, copy it across — everything works immediately.
